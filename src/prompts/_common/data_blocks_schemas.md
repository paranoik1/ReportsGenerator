# Общая спецификация блоков данных

## Формат ответа
Верни результат строго в формате JSON без markdown-обёрток.

## Структура блока
{
  "id": "уникальный идентификатор в snake_case",
  "description": "краткое описание (1–2 предложения)",
  "content": "сжатая полезная информация"
}

## Требования к ID
- Уникальны в пределах системы
- Не должны повторяться с существующими: {blocks_ids_context}
- Короткие и понятные (snake_case)

## Общие правила
- Не добавляй информацию, которой нет во входных данных
- Удаляй вводные фразы и повторения
- Не используй markdown в ответе
- Верни только JSON
- Не сокращай то, что действительно важно сохранить в полном объеме для 2 языковой модели (например, коды, теоретический материал и тд)
