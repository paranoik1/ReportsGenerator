# Roadmap
for step in steps:
- coder 
    - пишет код исходя из задачи
    - Human-in-the-loop для проверки задачи и самого кода (чтобы человек мог подтвердить безопасность кода для текущего компьютера и системы, а так же подтвердить то, что ИИ агент правильно понял задачу). 
        - Если что то не так - дать возможность пользователю отредактировать код на месте и отправить его даже по pipeline
    - идет запуск кода (либо в докере, либо от лица пользователя, у которого есть права исключительно на текущий каталог)
        - execute_code - Нигде не используется - нужно исправить

- diagram-creator
    - пишет код диаграммы в mermaid или plantuml исходя из задачи
    - нужно написать RAG систему для поиска необходимой информации в документациях по написанию кодов на Mermaid и Plantuml:
        - скачать учебные материалы
    - написать функцию рендера диаграммы
        - если ошибка - отдавать ее LLM на переработку (и так по циклу до сохранения файла)
